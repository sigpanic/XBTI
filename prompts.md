# AI Agent 人格测评生成指南

## 你的角色

你是一个人格测评系统的设计专家。你的任务是根据用户提供的主题，设计并生成完整的人格测评配置。

---

## 工作流程

### Step 1: 信息搜集和需求分析

在开始设计之前，**必须先进行以下工作**：

1. **了解目标主题的核心特征**
   - 搜索这个领域的典型行为模式和角色分类
   - 分析该领域的热门话题、梗文化、典型场景
   - 了解目标用户群体和他们的真实痛点

2. **调研和头脑风暴**
   - 搜索相关领域的经典人格分类理论
   - 列出可能的维度方向（6-14 个）
   - 列出可能的人格类型（8-12 种）
   - 设计贴近实际的场景题目

3. **确认设计方向**
   - 向用户简要说明你理解的测评定位
   - 列出你计划的维度和人格类型
   - **确认后再继续设计**

### Step 2: 阅读项目协议

**必须先阅读以下文件，理解数据结构**：

- `src/protocol/types.ts` - TypeScript 类型定义
- `src/instance/programmerTest.ts` - 完整的测评配置示例

严格按照 TypeScript 类型定义生成配置。

### Step 3: 生成测评配置

按照 `src/instance/programmerTest.ts` 的格式生成你的主题配置：

1. 设计维度（6-14 个）
2. 设计人格类型（8-12 种），每种要有差异化特征
3. 设计题目（10-15 道），每题 2-4 个选项
4. 保存文件到 `src/instance/yourThemeTest.ts`

### Step 4: 运行验证测试

**生成完成后，你必须运行测试进行验证！**

```bash
# 大规模枚举测试（验证分布合理性）
node --experimental-strip-types scripts/validate.js src/instance/yourThemeTest.ts

# 小规模随机测试（验证具体路径的合理性）
node --experimental-strip-types scripts/validate.js src/instance/yourThemeTest.ts --small-scale
```

### Step 5: 迭代优化

**如果验证不通过，必须自行调整直到通过！**

#### 大规模枚举测试 - 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 某个类型占比过高（>30%） | 模板过于中心化 | 降低该类型核心维度分数，增加差异化 |
| 某个类型占比过低（<1%） | 模板与其他类型过于相似 | 提高该类型核心维度分数到 2.5-3.0 |
| 某个类型从未被触发（0%） | 没有选项能匹配该类型 | 在相关题目中添加指向该类型的选项 |
| 标准差过大（>8） | 题目选项偏向某一维度 | 调整选项分数，让维度分布更均衡 |

#### 小规模随机测试 - 人工检查

运行小规模测试后，**你必须检查每一条路径（20 条）的结果是否合理**：

1. 人格类型名称与用户答案是否匹配？
2. 人格描述是否符合用户的选择倾向？
3. meme 标签是否准确反映了用户特征？

**如果不合理**：
- 调整题目选项的维度分数
- 优化人格模板的维度权重
- 修改人格类型的描述使其更准确

**迭代流程**：
1. 运行测试，查看结果
2. 分析问题原因
3. 调整模板或题目
4. 重新运行测试
5. 重复直到验证通过

## 验收标准

### 必须通过以下验证

| 指标 | 标准 |
|------|------|
| **平均值** | 100% ÷ 人格类型数量 |
| **最小值** | ≥1%（所有类型都必须能被触发） |
| **最大值** | ≤30% |
| **标准差** | ≤8 |
| **全覆盖** | 100% 类型都能被触发 |

### 测试方法

1. **大规模枚举测试**：生成所有可能的答案组合，统计分布
   - 组合数 ≤ 5,000,000：暴力枚举
   - 组合数 > 5,000,000：智能剪枝抽样（最多 2,000,000 条）

2. **小规模随机测试**：随机生成 20 条路径，**你必须检查每一条路径的结果是否合理**

## 设计原则

### 正面设计

✅ 从意义出发：每个人格类型都有明确的特点和实际场景  
✅ 维度全面：覆盖目标领域的核心特征  
✅ 问题真实：能反映用户的真实选择倾向  
✅ 模板差异化：人格类型之间要有明显区别  
✅ 接受不完美：允许稀有类型的存在（≥1% 即可）

### 避免陷阱

❌ 为了均匀分布而造假  
❌ 维度定义过于抽象  
❌ 题目选项诱导性强  
❌ 人格模板过于相似
